## Commands

```bash
# Install MkDocs Material (one-time)
pip install mkdocs-material

# Local development server with live reload
mkdocs serve

# Build static site (output: site/)
mkdocs build
```

The `site/` directory is gitignored — it's generated by CI, not committed.

## Version Numbering (PROJECT-RULES.md Rule 1)

**Every session: check and bump the version if behavior-changing commits were made.**

- **Single source of truth:** `VERSION` file at repo root
- **Displayed in:** Footer via `copyright` field in `mkdocs.yml` — update both when bumping
- **Current version:** `v0.6.0 beta`
- **Format:** `v{MAJOR}.{MINOR}.{PATCH} {stage}` — see `/Users/wmestrinho/.openclaw/workspace/PROJECT-RULES.md` for full rules

| Change | Bump |
|--------|------|
| Bug fix, copy tweak, style | PATCH |
| New section, new page, new feature | MINOR |
| Full restructure, new architecture | MAJOR |

Commit message must reference the version: `feat: add X — v0.6.0 beta`

## Deployment

**Cloudflare Pages** handles CI/CD — no GitHub Actions workflow needed.

- Triggers automatically on every push to `main`
- Build command: `pip install mkdocs-material && mkdocs build`
- Build output directory: `site`
- Python version: `3.11` (set via `.python-version` file)
- Custom domain: `plan.robotfantome.com` (DNS managed by Cloudflare)

The `site/` directory is gitignored — built by Cloudflare on each deploy, never committed.

## Content Conventions

- Use MkDocs admonitions (`!!! note`, `!!! warning`) for callouts
- Task lists use `- [ ]` / `- [x]` syntax (rendered via `pymdownx.tasklist`) — **interactive via checklist.js**
- Code blocks support annotations via `pymdownx.superfences`
- All headings get anchor links via `toc: permalink: true`
- Internal documents get `<span class="internal-badge">Internal Document</span>` badge
- Collapsible sections use `??? warning "Section Title"` for sensitive content
