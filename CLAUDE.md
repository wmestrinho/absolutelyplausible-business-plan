# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What This Repo Is

Living business plan for **Robot Fantôme / Absolutely Plausible** — a freelance agency umbrella. The content (strategy, timeline, brand, client work) is published as a static documentation site via MkDocs Material, auto-deployed to GitHub Pages on every push to `main`.

Deployed at: `https://wmestrinho.github.io/absolutelyplausible-business-plan/`

## Commands

```bash
# Install MkDocs Material (one-time)
pip install mkdocs-material

# Local development server with live reload
mkdocs serve

# Build static site (output: site/)
mkdocs build
```

The `site/` directory is gitignored — it's generated by CI, not committed.

## Site Structure

Navigation is defined in `mkdocs.yml`. Content lives in `docs/`:

```
docs/
  index.md           # Home — overview, financials, current focus checklist
  identity.md        # Who we are
  brand.md           # Brand reference (links to BRAND.md)
  strategy.md        # Full strategy doc
  timeline.md        # 6-month timeline
  stylesheets/
    extra.css        # Custom cyberpunk theme overrides for MkDocs Material
  projects/
    index.md         # Projects overview
    hackfatura.md    # hackFatura project for PCJ client
    roadmap.md       # Weekly checklist / task tracker
  clients/
    pcj.md           # PCJ / Parazinho Chassis Jig client page
```

`docs/` is the **single source of truth** for all content. Root-level source files no longer exist — edit directly in `docs/` and the site will reflect the change on next deploy.

## Deployment

GitHub Actions (`.github/workflows/deploy.yml`) handles CI/CD:
- Triggers on push to `main`
- Installs `mkdocs-material`, runs `mkdocs build`, deploys `site/` to GitHub Pages

The `jekyll-gh-pages.yml` workflow is vestigial — `deploy.yml` is the active one.

## Brand Rules

Two distinct brands — never mix them:

| | Robot Fantôme (us) | hackFatura / PCJ (client) |
|---|---|---|
| Background | White / light | Black `#0d0d0d` |
| Accent | Monochrome | Orange `#ff6d00`, Red `#ff003c` |
| Fonts | Serif headings, clean sans | Orbitron, Share Tech Mono |

The MkDocs site itself uses the **cyberpunk orange theme** (defined in `docs/stylesheets/extra.css`), not the Robot Fantôme clean/white aesthetic — this is intentional for the business plan presentation.

## Content Conventions

- Use MkDocs admonitions (`!!! note`, `!!! warning`) for callouts — styled with cyberpunk borders in `extra.css`
- Task lists use `- [ ]` / `- [x]` syntax (rendered via `pymdownx.tasklist`)
- Code blocks support annotations via `pymdownx.superfences`
- All headings get anchor links via `toc: permalink: true`
